# 📊 Инструкция пользователя QA Dashboard

Этот инструмент представляет собой веб-интерфейс (дашборд) для анализа качества кода и тестирования оборудования. Он объединяет в себе три функции:
1. **Heatmap Ошибок** — визуализация логов тестирования регистров.
2. **Оценка Pylint** — статический анализ качества Python-кода.
3. **Покрытие (Coverage)** — запуск тестов и проверка процента покрытия кода.

##  1. Подготовка окружения

Перед запуском убедитесь, что у вас установлен Python и необходимые библиотеки.

### Установка зависимостей
Откройте терминал (командную строку) и выполните команду:

```bash
pip install streamlit networkx pylint pytest pytest-cov
```

*   `streamlit`: движок для веб-интерфейса.
*   `networkx`: используется для внутренней структуры данных.
*   `pylint`: анализатор кода.
*   `pytest` и `pytest-cov`: инструменты для тестирования и расчета покрытия.

### Размещение файлов
Для корректной работы программы убедитесь, что в одной папке со скриптом `Dashboard.py` находятся рабочие файлы:
1. **`uart_final_summary.txt`** — файл с логами ошибок (для Heatmap).
2. **`golden_tester.py`** — Python-скрипт или тестбенч, который вы хотите проверить (для Pylint и Coverage).

> *Примечание: Если этих файлов нет, программа создаст демонстрационные файлы-заглушки при первом запуске.*

##  2. Запуск Дашборда

Чтобы открыть программу, выполните следующую команду в терминале, находясь в папке с проектом:

```bash
python -m streamlit run Dashboard.py
```

После выполнения этой команды ваш браузер автоматически откроет вкладку с адресом `http://localhost:8501`. Если этого не произошло, скопируйте ссылку из терминала и вставьте её в браузер вручную.

##  3. Как пользоваться интерфейсом

Слева находится **Боковая панель (Навигация)**, где можно переключаться между тремя режимами работы.

### Вкладка 1:  Heatmap Ошибок
Используется для визуального анализа ошибок, найденных в регистрах (данные берутся из `uart_final_summary.txt`).

*   **Таблица (Сетка):**
    *   Строки — адреса регистров.
    *   Столбцы — типы ошибок (RO Denied, Mismatch и т.д.).
    *   **Цвет ячейки:** Чем ярче цвет, тем чаще встречалась ошибка. Зеленый цвет — для ошибок доступа (RO Denied), Красный — для остальных ошибок.
    *   **Подсказки:** Наведите курсор на ячейку, чтобы увидеть точное количество ошибок.
*   **Сводная статистика:** Нажмите на стрелочку "Сводная статистика", чтобы развернуть и просмотреть текстовую часть лога.

### Вкладка 2: Оценка Pylint
Используется для проверки стиля кода и поиска синтаксических ошибок в файле `golden_tester.py`.

1.  Нажмите большую кнопку **"ЗАПУСТИТЬ PYLINT"**.
2.  Дождитесь окончания анализа (индикатор загрузки сверху).
3.  **Результат:**
    *   Появится крупная оценка вашего кода по 10-балльной шкале.
4.  **Детали:** Нажмите "Показать полный лог", чтобы увидеть список конкретных замечаний (где пропущены пробелы, неверные имена переменных и т.д.).

### Вкладка 3: Покрытие
Используется для запуска модульных тестов (`pytest`) и проверки того, какой процент кода покрыт тестами.

1.  Нажмите большую кнопку **"ЗАПУСТИТЬ PYTEST (COV)"**.
2.  Программа запустит тесты для файла `golden_tester.py` с флагом `--cov`.
3.  **Результат:**
    *   Появится круговая диаграмма с процентом покрытия.
    *   **Цветовая индикация:**
        *   🔴 **Красный:** Покрытие < 50% (Плохо).
        *   🟡 **Желтый:** Покрытие от 50% до 80% (Средне).
        *   🟢 **Зеленый:** Покрытие > 80% (Отлично).
4.  **Детали:** Нажмите "Показать полный лог тестов", чтобы увидеть результаты прохождения тестов (PASS/FAIL) и детальный отчет по строкам.

## ❓ Возможные проблемы

1.  **Ошибка "ModuleNotFoundError: No module named..."**
    *   Вы забыли установить одну из библиотек. Повторите шаг 1 "Установка зависимостей".
    *   Для вкладки "Покрытие" критически важно наличие `pytest-cov`.

2.  **Дашборд показывает демо-данные вместо реальных**
    *   Проверьте, что вы положили ваши файлы `uart_final_summary.txt` и `golden_tester.py` **рядом** с файлом `Dashboard.py`.
    *   Если программа уже создала файлы-заглушки, удалите их или замените своим содержимым.

3.  **Кнопка "Покрытие" выдает предупреждение**
    *   Убедитесь, что в файле `golden_tester.py` есть корректные тесты или он совместим с запуском через `pytest`.
